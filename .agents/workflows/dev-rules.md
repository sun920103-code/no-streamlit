# 量化 Python + FastAPI + Vue3 极简避坑 20 条

> 改动最小、不废旧代码

## 一、后端 FastAPI 避坑（10 条）

1. **核心计算一行不改** — 只包接口，不动 HRP/BL/选基逻辑
2. **全部加超时（防卡死）** — 任何 Wind、矩阵、回测都设 `timeout=15s`
3. **禁止全局变量存大 DataFrame** — 否则内存爆、越跑越卡
4. **路径全部写绝对路径** — 相对路径本地能跑、上线必挂
5. **Wind / 数据库用完立刻 close** — 不释放 → 越请求越堵、最后崩
6. **所有异常兜底返回 JSON** — 不让后端报错直接抛栈 → 前端空白
7. **DF 只转 dict/list 出去** — 不要传 object、不要传图对象
8. **接口只做分发，不做重计算** — 重算丢后台任务，避免超时
9. **开启跨域 CORS** — 90% 联调不通都是这个
10. **加日志、记录入参 + 耗时** — 出问题一秒定位

## 二、前后联调避坑（5 条）

11. **先定字段，谁都不许乱改** — 一改前端全崩
12. **后端只吐纯净 JSON（表、数组、数值）** — 不吐图表、不吐打印、不吐文本乱码
13. **状态不要放前端内存** — 本地变量习惯，搬到线上全丢
14. **分页、数量、排序前后对齐** — 不然数据"少了/不见了"
15. **错误信息透到前端** — 不要默默失败

## 三、Vue3 少踩坑（5 条）

16. **变量必须 ref/reactive** — 忘了 → 页面不刷新、像丢功能
17. **不要硬迁旧 Vue2 写法** — 成本极高、容易心态崩
18. **图表、表格交给前端渲染** — 后端不画图（画了也看不见）
19. **生命周期不要乱绑** — 接口请求时机错 = 数据空
20. **组件拆分别太急** — 先跑通，再优化，别一边改一边烂
