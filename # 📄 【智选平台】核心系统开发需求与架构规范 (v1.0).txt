# 📄 【智选平台】核心系统开发需求与架构规范 (v1.0)

## 🚨 第一部分：系统级架构红线（开发必读）
本系统（智选平台）为独立的高净值客户资产配置系统。为绝对防止底层 C 语言数学库（OpenBLAS）多线程冲突及业务数据污染，开发必须严格遵守以下红线：
1. **进程与变量物理隔离**：本系统前端状态变量必须带上专属前缀 `zx_`。严禁与历史旧系统混用任何持仓池或缓存变量。
2. **底层引擎切割**：本系统的计算标的为 **114 支核心精选基金**。所有收益率、协方差矩阵计算引擎，必须封装为独立的 API 接口，严禁在前端直接跑重度算力。
3. **架构强制要求**：前后端解耦。Streamlit 仅作 UI 渲染与 HTTP 请求发送，核心计算全量下沉至 FastAPI 后端。

---

## 🧭 第二部分：全局交互与导航栏规范
左侧导航栏（Sidebar）作为全局控制中枢，包含以下参数输入区：
* **配置资金**：数值输入，单位“万元”（默认 1000 万）。
* **预期年化收益率**：数值输入，单位“%”。
* **最大年化波动率上限**：数值输入，单位“%”。
* **预计持仓时间**：下拉框，选项【半年、1年、3年】（作为后续 KPI 测算基准）。
* **核心精选基金池**：抽屉式展示 114 支基金（按权益、固收、现金分类）。**注：现金管理产品年化波动率强制硬编码为 0。**

## 🏗️ 第三部分：核心功能页面细则

### 页面一：【宏观配置底仓】
**核心业务逻辑**：
1. **获取宏观象限**：对接 EDB 宏观因子，输出当前宏观周期象限。
2. **测算分支规则**：
   * **情景 A（波动率满足要求）**：生成三套方案（进取：收益+20%；稳健：基准收益；防守：收益-20%）。
   * **情景 B（波动率无法满足）**：强制降级，以用户输入的波动率为绝对锚点，仅生成一套【稳健配置】。
3. **输出要求**：需渲染 KPI 卡片组（收益、波动、最大回撤、夏普），及对应资产明细权重/金额表格。

### 页面二：【战术配置】
**核心业务逻辑**：
1. 基于【宏观配置底仓】中的**稳健配置**进行战术偏移。
2. 支持两条管线：**新闻资讯自动调仓** 或 **专属研报上传调仓**。
3. **输出要求**：
   * 并排展示 3 组 KPI 对比（原稳健底仓 vs 资讯调仓 vs 研报调仓）。
   * 输出调仓明细表（展示增减偏移比例及金额）。
   * **MoE 投委会分析报告**：调用大模型，生成 500 字以内的调仓逻辑与决策依据。

### 页面三：【业绩回测】
**核心业务逻辑**：
1. 展示配置方案与主流宽基指数过去 5 年的绝对回报率对比。
2. **图表视觉规范**：柱状图，数据标签上标百分比，**严格执行“红涨绿跌”着色**。
3. **基准池**：上证指数、沪深300、深证成指、中证500、中证1000、创业板指、恒生指数。

## 🔌 附录：FastAPI 后端核心接口规划 (Contract)
开发团队请依据此清单搭建后端 `No Streamlit` 算力引擎：

1. **宏观底仓引擎**
   * **Endpoint**: `POST /api/v1/macro_allocation`
   * **Payload**: `{ "capital": 1000, "target_ret": 8.0, "max_vol": 15.0, "period": "1年" }`
   * **Response**: 返回 1 套或 3 套配置的具体权重分配与 4 项 KPI。

2. **研报/资讯战术引擎**
   * **Endpoint**: `POST /api/v1/tactical_adjustment`
   * **Payload**: `{ "base_allocation": {...}, "report_file": (optional file) }`
   * **Response**: 返回调仓偏移后的权重表、新 KPI 及 MoE 大模型的 500 字分析依据。

3. **历史回测引擎**
   * **Endpoint**: `POST /api/v1/backtest`
   * **Payload**: `{ "allocation_weights": {...}, "benchmarks": ["沪深300", "上证指数"] }`
   * **Response**: 返回过去 5 年年度收益率时间序列数据（供前端绘图）。
